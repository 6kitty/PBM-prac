## 구조

```
src/
├── constraints/
│   ├── schema.py            # 6종 Constraint Pydantic 모델 + EvaluationContext/Result
│   ├── registry.py          # ConstraintRegistry 팩토리 + evaluate_pre/post
│   └── evaluators/
│       ├── pre.py           # MaxDailySpend, MaxPerTx, TimeWindow, AllowedPlatforms
│       └── post.py          # MaxCPA, MinCTR
├── mandate/
│   ├── client.py            # Ed25519 JWS-style sign/verify (AP2 SDK 동등 wrapper)
│   └── factory.py           # Open/Closed Mandate 빌더
├── x402/
│   ├── server.py            # Resource Server (가맹점 mock, 402 발행)
│   ├── facilitator.py       # Facilitator: verify/settle/kpi_push/post_eval
│   └── client.py            # Agent X-PAYMENT 빌더
└── policy_server.py         # MandateRuntimeState + run_pre/run_post

tests/
├── unit/
│   ├── test_pre_evaluators.py    # PRE 4종 × 정상/경계/위반
│   └── test_post_evaluators.py   # POST 2종 × 표본부족/임계/위반
└── integration/
    ├── test_happy_path.py
    ├── test_failure_scenarios.py # 실패 7종 (서명/만료/PRE/일치성/network/POST/nonce)
    └── test_open_vs_closed_baseline.py
```

## 변인통제

- 거시(Mandate constraints) ↔ 미시(paymentRequirements) 하이브리드
- Facilitator = x402 게이트웨이 + 정책 서버 (한 신뢰 경계 안)
- Mandate=User 서명, Receipt=Facilitator 서명 (Ed25519)
- 모든 통신 localhost loopback / in-memory store
- 시간은 `requested_at` 명시 주입으로 제어
